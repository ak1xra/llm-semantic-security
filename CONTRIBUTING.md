# Contributing to llm-semantic-security

## Before You Open a PR

Open an Issue first.
All contributions start with a discussion — not a pull request.
If there is no Issue, the PR will be closed without review.

## What We Welcome

- New case studies (SIF diagnostic format required)
- Checklist refinements (include scenario + proposed adjustment)
- Translation (non-English versions of README or framework docs)
- Application domain extensions (healthcare AI, legal AI, autonomous systems, etc.)
- L4/L5 research connections (interpretability, activation steering)

## What We Do Not Accept

- Changes to core definitions (S5LA layer definitions, SIF layer definitions)
- Changes to Human-in-the-Point principle definition
- Changes to 3BPS architecture description
- Renaming or reframing of S5LA / SIF / SSF terminology
- PRs without a linked Issue

## Core Framework Is Frozen

S5LA v1.0 layer definitions are frozen.
SIF L1/L2/L3 definitions are fixed.
These are not open for revision via PR.

Application examples, case studies, and translations may be added.
Core definitions are not modified.

## PR Format

Title: [Type] Short description
Types: case-study / checklist / translation / research / docs

Body must include:

- Linked Issue number
- What changed and why
- What was NOT changed (scope boundary)

## Decision Authority

All merge decisions are made by the repository owner.
PRs that pass review do not merge automatically.
Explicit approval is required for every merge.
